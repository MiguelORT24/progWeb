1.	Introducción

1.1	Propósito.
Control de inventario y administración de servicios de instalación incluyendo definición en detalle de los requerimientos del sistema web para la gestión integral de un almacén de cámaras, sensores y componentes electrónicos.

1.2	Alcance.
El sistema permitirá:
•	Registrar y consultar inventario por lotes, ubicaciones y estados.
•	Gestionar compras a proveedores y movimientos de inventario.
•	Administrar órdenes de instalación y su trazabilidad con el stock.
•	Control de acceso por roles (ADMIN, ALMACEN, LECTOR).
•	Búsqueda y filtrado avanzados del catálogo.

1.3	Definiciones, siglas y abreviaturas.
•	Equipo: ítem de inventario (cámara o sensor).
•	Lote: agrupación de unidades de un mismo equipo.
•	Movimiento: entrada, salida o ajuste sobre un lote.

1.4	Referencias.
•	Documento base con objetivos, entidades y relaciones.
•	IEEE Std 830-1998 y 1471.

1.5	Visión general del documento.
Se describen primero el contexto y usuarios; luego, los requisitos específicos funcionales y no funcionales, datos, interfaces, restricciones y criterios de aceptación.

1.6	Objetivos del sistema.
El sistema tiene como objetivos principales:
•	Generar reportes diarios: Producir un informe al final del día que detalle la cantidad de unidades por componente y su ubicación específica en el almacén.
•	Actualizar el inventario en tiempo real: Reflejar inmediatamente las entradas y salidas de todos los componentes para que la información de stock sea siempre precisa.
•	Permitir el registro de nuevos productos: Ofrecer la funcionalidad para que los usuarios puedan dar de alta nuevos tipos de productos o componentes en el catálogo del sistema.


2.	Descripción general

2.1	Aplicación web modular con:
•	Frontend: interfaz para consulta/operación de inventario y órdenes.
•	Backend: API para lógica de negocio y persistencia.
 
•	BD relacional: esquema con entidades equipo, marca, categoría, ubicación, inventario_lote, movimiento_inventario, proveedor, compra, compra_detalle, cliente, usuario y sus relaciones.
2.2	Funciones del producto (resumen)
•	ABM de catálogos (marcas, categorías, ubicaciones, equipos).
•	Gestión de stock por lotes y estados (DISPONIBLE, RESERVADO, INSTALADO, DAÑADO).
•	Registro de compras y movimientos de inventario.
•	Órdenes de instalación con reserva/consumo de stock.
•	Reportes e historiales.
2.3	Usuarios y características
•	ADMIN: configuración global, seguridad, reportes completos.
•	ALMACEN: operaciones de inventario, compras y movimientos.
•	LECTOR: consulta e informes sin modificar datos.
2.4	Restricciones Generales
•	Datos críticos auditables (quién, cuándo, qué).
•	Controles de acceso por rol.
•	Respaldos periódicos de BD.
•	Escalabilidad para nuevas familias de productos.
•	Tecnologías de desarrollo: El sistema se implementará utilizando HTML, CSS y JavaScript para la capa de presentación, PHP como lenguaje de servidor, y una base de datos relacional MySQL.
2.5	Suposiciones y dependencias
•	Autenticación por email y contraseña.
El sistema gestionará usuarios con credenciales únicas (email/contraseña), almacenadas de manera segura. No se contempla integración con inicio de sesión único (SSO) en esta versión.
•	Infraestructura web estándar.
El sistema está diseñado para ejecutarse en entornos web convencionales (servidores locales o en la nube), utilizando tecnologías ampliamente soportadas (ej. servidores Linux/Windows con base de datos relacional).
•	Compatibilidad con navegadores modernos.
Se prevé compatibilidad con navegadores ampliamente utilizados como Google Chrome, Mozilla Firefox y Microsoft Edge en sus versiones recientes.
